## クイックスタート

### 初めての方

1. [セットアップガイド](./setup.md)を読んで環境を構築
2. [アーキテクチャ](./architecture.md)を読んで設計を理解
3. [開発ガイド](./development.md)を参照して実装開始

### すでに開発している方

- 新機能追加: [開発ガイド - 新機能の追加手順](./development.md#新機能の追加手順)
- トラブル対応: [セットアップガイド - トラブルシューティング](./setup.md#トラブルシューティング)
- コーディング規約確認: [開発ガイド - コーディング規約](./development.md#コーディング規約)

## プロジェクト構成

```
apps/api/
├── config/          # アプリケーション設定
├── controller/      # HTTPコントローラー
├── db/              # データベース接続
├── docs/            # このドキュメント
├── gateway/         # リポジトリ実装（インフラ層）
├── middleware/      # HTTPミドルウェア
├── migrate/         # データベースマイグレーション
├── model/           # ドメインモデル
├── repository/      # リポジトリインターフェース
├── router/          # ルーティング設定
├── usecase/         # ビジネスロジック
├── util/            # ユーティリティ関数
└── main.go          # エントリーポイント
```

詳細は[アーキテクチャドキュメント](./architecture.md)を参照してください。

## 技術スタック

- **言語**: Go 1.25.2
- **Webフレームワーク**: Echo v4
- **ORM**: GORM
- **データベース**: PostgreSQL 16
- **スキーマ共有**: OpenAPI 3.0

## コントリビューション

プロジェクトに貢献する際は、以下を確認してください:

1. [開発ガイド](./development.md)のコーディング規約に従う
2. 適切なテストを書く
3. ドキュメントを更新する（必要に応じて）

問題が発生した場合:

1. [セットアップガイドのトラブルシューティング](./setup.md#トラブルシューティング)を確認
2. [開発ガイドのよくある質問](./development.md#よくある質問)を確認
3. それでも解決しない場合はチームに相談
